# AGENTS.md

This repo is part of Works Lab.

## Voice

Write plainly. Avoid hype, generic startup language, and AI-sounding filler. Prefer practical descriptions of what exists, what is planned, and what is still unknown.

## Project status

This project is early. Do not describe prototypes, hardware, firmware, measurements, or performance as finished unless files in the repo prove it.

## Documentation rules

- Keep READMEs short and useful.
- Put longer explanations in docs/.
- Mark assumptions clearly.
- Use "planned," "proposed," "draft," or "TBD" when appropriate.
- Do not introduce private or unrelated project references unless explicitly instructed.
- Do not invent licensing details.

## Validation

For documentation-only changes, check Markdown formatting manually. If tooling exists later, run the repo's documented checks before committing.
